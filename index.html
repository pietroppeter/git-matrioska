git-matrioska

a normcore git workflow

for maintaining production (ML) repos

pietroppeter

preamble

  • I work in Supply Chain B2B company

  • maintaining a production ML repo for past 7 years

  • problem: keeping a clean production repo...

  • ...while documenting (in a repo!) our maintenance

a repo inside a repo, the solution is...

git submodule

the idea

  1. start with a production (ML) repo (e.g. mliska)

  2. feel the problem (clean prod & track changes)

  3. create mloska

  4. checkout mliska inside mloska

  5. add mliska to .gitignore

Example folder structure

mloska/
├─ .git/
├─ dev/
│  ├─ 1_first_feature.ipynb
│  ├─ 2_first_bug.py
│  ├─ 3_not_a_bug_yay.md
├─ mliska/
│  ├─ .git/
│  ├─ app.py
├─ .gitignore
